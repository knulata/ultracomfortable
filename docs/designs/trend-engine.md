## Vision

### 10x Check
The current plan is "scrape trends → dashboard → WhatsApp blast → partners say bisa." That's a newsletter with AI. The 10x version is an **intelligent supply coordinator** — an AI agent that predicts trends before they peak, allocates production across partners to prevent glut, sets dynamic pricing, manages the full cycle from detection to payout, and creates a marketplace moat through the partner network + allocation intelligence.

Enterprise tools (Heuritech, Trendalytics) charge massive prices for trend detection alone. No one does supply coordination for Indonesian modest fashion SMEs. The same underlying AI capability now costs ~$50-100/day in API costs. The differentiation is domain specificity (Indonesian modest fashion) + distribution (WhatsApp to Tanah Abang) + coordination (intelligent allocation).

### Platonic Ideal
A Tanah Abang shop owner opens WhatsApp at 7 AM. They see: "Good morning, Bu Sari. Today we recommend you produce: (1) Sage green A-line gamis — trending +340% this week, 0 other partners producing, suggested retail Rp 189,000. (2) Pearl detail khimar — trending on TikTok and Shopee, 2 partners committed but demand gap is still 500 units." Bu Sari taps "Bisa" on both. The AI checks her past fulfillment rate (95%) and production capacity, confirms the allocation, and auto-generates a listing with trend-sourced photos + AI-written descriptions. By 9 AM, the product is live on alyanoor.com. By end of day, 3 orders come in. Bu Sari produces, ships, gets paid. She never opened Instagram.

The feeling: "Someone smarter than me is running my business strategy, and I just need to make the clothes."

## Key Decisions from Review

### Demand Model
Dual revenue: marketplace (alyanoor sells, partners consign) + SaaS intelligence (partners use data for their own channels). Commission on marketplace sales + subscription fee for data access.

### Implementation Approach
Full automated pipeline (Approach B from design doc) — scrapers + AI + dashboard + WA broadcast + photo matching + ordering. All 6 expansions in scope.

### Prerequisite Identified
The entire partner system is currently Zustand + localStorage. Must migrate to Supabase before building trend engine. This was not in the original design doc.

### Landscape Insight (Eureka)
The real product isn't trend detection — it's supply coordination with allocation. Enterprise tools (Heuritech, Trendalytics) do trend detection for $$$. No one does supply coordination for SMEs. The coordination layer + partner network is the defensible moat.

## Phasing

### Phase 1: Foundation (Week 1)
- Supabase migration (partner data from localStorage to real DB)
- 4-platform scrapers (IG, TikTok, Shopee, Tokopedia via Apify)
- AI vision pipeline (Claude Sonnet for style attribute extraction)
- Trend scoring + clustering
- Admin trend dashboard

### Phase 2: Distribution (Week 2)
- WhatsApp Business API integration (start application on Day 1)
- Daily trend broadcast to partners
- Partner "bisa" response tracking
- Partner dashboard (trending styles + commit button)
- Live viral trend alerts

### Phase 3: Intelligence (Week 3)
- Intelligent production allocation
- Trend prediction (velocity curve analysis)
- Photo matching + fabric-to-trend matching
- Auto-generated product listings
- Partner scorecard + seasonal forecasting
- 500/day tracker + gap analysis

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Scrapers break (platform changes) | High | High | Use Apify managed actors (they maintain); build fallback to manual curation |
| WA Business API approval delayed | Medium | High | Start application Day 1; fallback to WA group broadcast |
| Trend prediction accuracy low | Medium | Medium | Start with detection only; prediction improves as historical data accumulates |
| AI vision misclassifies styles | Low | Medium | Human review loop for top-20 daily trends; improve prompts over time |
| Partner adoption slow | Medium | High | The Assignment: validate with 3 real partners before building |
| Scraped photos have copyright issues | Medium | Medium | Auto-generated listings use AI mockups, not scraped photos directly |
| API costs exceed budget | Low | Medium | Use Sonnet (not Opus) for vision; batch processing; cache embeddings |
